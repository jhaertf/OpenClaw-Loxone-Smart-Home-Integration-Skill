# Datenquellen & Sensitivität

- Alle IDs/UUIDs im Repo sind Platzhalter (`REPLACE_WITH_UUID_*`).
- Secrets gehören nicht ins Repo: nutze Credential-Dateien oder Environment.
- Host/User/Passwort immer über `config.json` bzw. Env injizieren.
- In Nutzerantworten nur semantische Namen + Ergebnis, keine Roh-IDs.
